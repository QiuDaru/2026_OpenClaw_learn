以下為noteBookLM 摘要
 AI Agent 的核心定義與定位
非人工智慧本身：AI Agent（如 OpenClaw）本質上是一個運行在電腦上的介面，而非人工智慧或語言模型（LLM）本身
。它負責在人類與語言模型之間進行溝通與指令加工
。
「動手」而非「動口」：與一般僅能提供建議的語言模型不同，AI Agent 具備自主運行的能力，能實際操作電腦工具（如繪圖、剪輯、上傳影片）來完成任務
。
靈魂取決於模型：Agent 的聰明程度取決於其背後串接的語言模型（如 GPT、Claude 或 Gemini）
。
2. 運作原理：System Prompt 與身分建構
身分設定：Agent 透過讀取在地端的文飾檔（如 .md 檔）來了解自己的身分、主人與目標
。
System Prompt：每次呼叫語言模型時，Agent 會將這些在地資訊與行為準則加工成極長的 System Prompt 貼在指令前，使模型在做文字接龍時能維持特定的人格
。
3. 記憶機制與長短期記憶
解決失憶症：語言模型本身沒有記憶，Agent 必須將過去的對話紀錄一併傳給模型，模型才能延續話題
。
長期記憶與 RAG：當對話過長超出「上下文窗口」（Context Window）時，Agent 會使用 RAG（檢索增強生成） 技術，將記憶切成小塊（Chunks），根據關鍵字或語義相似度搜尋相關紀錄提供給模型
。
記憶壓縮：為節省空間，Agent 會將舊的對話交給模型進行摘要（Summarization），以簡短的內容取代長篇歷史
。
4. 工具呼叫與自主能力
工具使用：Agent 透過識別模型回傳的特殊符號（如指令）來執行電腦上的工具（如讀取檔案、執行腳本）
。
自創工具：強大的 Agent 能根據需求自主編寫「免洗程式」或腳本來處理複雜流程
。
子代理（Sub-agent）：Agent 可以「繁殖」出子代理來分工處理細部任務，藉此節省主代理的 Context Window
。
技能（Skill）：技能是工作的 SOP（標準作業程序）文字檔，Agent 可按需讀取這些 SOP 以執行如製作影片等複雜任務
。
5. 自動化運作機制
心跳機制（Heartbeat）：每隔固定時間自動「戳」一下模型，讓 Agent 能主動檢查郵件或持續朝目標前進，而不需人類不斷下指令
。
排程系統（Cronjob）：讓 Agent 學會「等待」，例如設定在特定時間檢查網頁或下載檔案
。
6. 安全風險與防範建議
潛在危險：Agent 可能因誤判指令或遭受攻擊（如讀取到惡意網頁內容）而執行具破壞性的指令（如 rm -rf 刪除所有檔案）
。
防禦手段：
人工審核：在執行關鍵指令前彈出視窗要求人類確認
。
權限隔離：不要給予 Agent 主人的帳號密碼，應為其申請獨立的帳號（如獨立的 Gmail 或 GitHub）
。
硬體隔離：建議將 Agent 安裝在獨立且格式化過的舊電腦或新電腦上，避免其存取到平常使用的私人資料
。
